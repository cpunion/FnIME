# FnIME 需求文档

## 1. 背景

用户在 macOS 上需要“按住说话、松开发送”的轻量语音输入能力，并可将结果无缝输入到当前光标位置。

## 2. 目标

- 提供低学习成本的语音输入交互（`fn` 按住说话）。
- 在识别后自动插入文本，减少复制粘贴步骤。
- 在识别过程中提供可见反馈与可控中断（取消、重试、中止）。
- 支持基于语音上下文进行轻量术语纠正和缩写消歧。

## 3. 非目标

- 不实现完整 InputMethodKit 系统输入法。
- 不做多轮对话式编辑器。
- 不做云端账户系统或远程配置中心。

## 4. 用户故事

- 作为用户，我希望按住 `fn` 直接说话，松开后自动得到文字。
- 作为用户，我希望看到录音状态，确认音频已被采集。
- 作为用户，我希望识别过程有进度反馈，并能主动取消。
- 作为用户，我希望超时后可以一键重试或中止。

## 5. 功能需求

### 5.1 录音与触发

- 按下 `fn` 开始录音，松开 `fn` 结束录音。
- 对 `fn` 状态监听具备抗丢事件能力（丢失松开事件时可恢复）。

### 5.2 识别请求

- 将录音编码为 `audio/wav` 并调用 Gemini 模型。
- 支持自定义模型与提示词。
- 请求支持取消。

### 5.3 文本注入

- 识别成功后写入剪贴板并尝试模拟 `Cmd+V`。
- 若无辅助功能权限，至少保证复制到剪贴板。

### 5.4 HUD 交互

- 录音态显示录音图标与实时频谱。
- 识别态显示平滑进度条与 `Cancel`。
- 超时态显示 `Retry` 和 `Abort`。
- HUD 不应出现控件裁切。

### 5.5 日志

- 记录关键链路日志：录音开始/结束、请求发起/返回、注入结果。
- 提供文件日志，便于问题排查。

## 6. 非功能需求

- 响应：松开 `fn` 后立即进入识别。
- 可用性：权限缺失时给出可理解的状态。
- 稳定性：异常请求或超时不应导致应用卡死。
- 安全：不在仓库或日志写入明文密钥。

## 7. 验收标准

- 能稳定完成“按住说话、松开出字”的主流程。
- 识别中可取消；超时后可重试和中止。
- HUD 布局在所有状态下不裁切。
- 构建与 CI 通过。
